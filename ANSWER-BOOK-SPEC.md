# 答案之书 — 功能规格文档

## 概念

泡沫来信的第二个功能模块：**答案之书**。

用户写下一个问题或一段心情 → 屏幕上浮出一个词（大字，带动画）→ 下方出现一两句话，是"触不到的恋人"用这个词回应你。

不是算命，不是随机生成器。是**有个人在替你想**。

---

## 用户流程

1. **入口**：首页新增入口（和"拆一封信"并列），按钮文案待定，参考："问一个问题" / "翻一页答案"
2. **提问屏**：用户看到一个输入框，可以打字（问题或叙述都行），也可以留空直接翻
3. **翻书动画**：点击后，进入翻书/翻页动画（类似旧书翻页，或卡片翻转）
4. **答案展示屏**：
   - 一个大字词语居中出现（带浮入动画，从模糊到清晰）
   - 下方一两句话解释，是AI以"触不到的恋人"身份写的
   - 配图（先用图库，后期可接生图API）
5. **底部按钮**："再问一次" / "保存答案" / 返回首页

---

## 词库

使用泡沫来信现有词库（`js/bubbles.js` 的 `WORD_POOL`），从中随机抽取一个词作为"答案"。

词库分类：情绪 / 氛围 / 行为 / 梦核 / 真理沙滩 / 英文

也可以额外上传专属答案词库（更适合"答案"语境的词），格式同现有词库。

## AI Prompt

调用现有的 `/api/letter` 接口，或新建 `/api/answer` 接口。

**System prompt:**
```
你是一个触不到的恋人。你只能通过一个词和一两句话回应对方。
```

**User prompt 结构:**
```
对方问了你一个问题（也可能只是说了一段心情）：
"{用户输入的文字}"

你翻开答案之书，落在了这个词上：
「{随机抽到的词}」

用1-2句话，以这个词为锚点，回应对方。

规则：
- 最多两句话，40字以内。每个字都要有意义。
- 语气温柔但克制，像隔着很远的地方轻轻说了一句话。
- 不要解释这个词的字面意思，而是用它来回应对方的处境和情绪。
- 如果用户没有输入文字（留空），就单纯地说说你看到这个词时想到了什么。
- 不要用"亲爱的"。不要署名。
- 禁词：接住、涟漪、石子、泛起。
- 不要提"见过你"。
- 全文中文。
```

**模型**：GPT-5.4 / Claude-sonnet-4-6 随机轮换（和来信一样）

## UI/UX 设计规范

### ⚠️ 必须保持的视觉风格

泡沫来信已有一套完整的视觉语言，答案之书必须**完全继承**，不要另起炉灶：

1. **双主题系统**（CSS变量驱动，`data-theme="light"` / `data-theme="dark"`）
   - 白色晨雾主题：`--bg-deep: #e8ecf1`，冰蓝/藤紫/雪白极光
   - 暗色星雾静蓝主题：`--bg-deep: #050606`，天渊蓝 `#04345a` 极光，雾银文字 `#c8c6c6`
   - 暗色模式有呼吸式星光canvas（`#starfield`）

2. **毛玻璃（Glassmorphism）**
   - 所有卡片/面板：`backdrop-filter: blur(16px) saturate(1.2)`
   - 背景半透明，边框极淡
   - `var(--glass-bg)` / `var(--glass-border)` 变量

3. **按钮风格**
   - 渐变背景 `linear-gradient(135deg, ...)`
   - `inset` 顶部高光线
   - `letter-spacing: 0.1em`
   - hover 时用 `var(--accent)` 发光
   - `border-radius: 24px`

4. **字体**
   - 界面：`'Noto Serif SC', serif`
   - 答案大字推荐：`'LXGW WenKai'` 或 `'Noto Serif SC'`
   - 解释文字：`'Noto Serif SC'` 细体

5. **动画**
   - 元素出现：blur(4px) → blur(0) + translateY 浮入
   - 缓动：`cubic-bezier(0.4, 0, 0.2, 1)`
   - 微交互：hover translateY(-2px)

6. **色彩**
   - **不要使用任何 hardcoded 颜色**，全部走 CSS 变量
   - 参考 `css/style.css` 中 `:root` 和 `[data-theme="dark"]` 的变量定义

### 答案展示屏布局参考

```
┌─────────────────────────┐

│                         │

```

## 配图方案

### Phase 1（先做）：图库
- 准备 30-50 张氛围图片，分类对应词库情绪分类
- 存在 `images/answers/` 目录
- 根据抽到的词的分类，随机选一张同类图片
- 图片风格：淡色/模糊/胶片感/自然/光影，不要二次元不要插画

### Phase 2（后做）：AI 生图
- 接生图 API，根据词+用户输入生成配图
- 风格 prompt 另写
